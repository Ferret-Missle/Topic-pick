# 【AI】トピックピック

## 概要

- 指定したワードや文章から、特定のトピックに関する最新情報を収集する。
- 複数のトピックを登録できるようにし、平行して情報を追跡する。
- 各トピックで、サマリーや更新ごとのトレンド、エコシステムの盛り上がり、関連・より良いトピックの提案・変更を提供

## 要求事項

1. PC・スマホで同一のデータを参照できること
2. トピックの精度を上げること
3. 複数トピックを管理できること。ひとまずMAX10個とする
4. 各トピックでサマリを表示できること
5. 各トピックでトレンド・エコシステムの規模などの市場情報を取得できること（グラフ）
6. トピックが趣旨から外れた場合、より良いトピックを提案・切替できること
7. 各トピックごとの情報取得の方針・抽出を確認できること（検索ワード）
8. 参照先を明示できること
9. Markdown形式でエクスポートできること

## 要件

#### トピック管理

1. AIとチャットで相談しながら追加するトピックを決定できる
2. トピックを削除できる
3. トピックは指定されたMAX数まで登録できる

#### トピック詳細

1. 定期的に自動で最新情報を取得できる
2. 取得頻度は基本的に週に１回、１トピックだけデイリー更新が可能
3. 課金で全部デイリー更新、かつ即時更新ボタンを実装
4. サマリをトピック詳細ページの上部に表示する
5. サマリはカードに記載すること
6. どういう条件でトピックを検索したかをサマリカードの下に折り畳み可能な形式で表示する
7. トピックのトレンドや盛り上がり、トピックの関心の規模などをカードに表示
8. サマリにする前の情報を一覧で列挙し、折り畳み可能な形式で表示する

#### UX

1. トピックがユーザから必要とされていない場合、変更できる
2. トピック変更をAIに相談できる
3. トピックの最終閲覧時間と１週間・１か月間の閲覧回数を保持
4. 関心度が低いと判断した場合は、トピックの変更提案を行う

### UI

1. レスポンシブ対応
2. 左ペインにトピック一覧を表示
3. デイリー更新の対象トピックはマークを付与すること
4. 右のメインペインは、サマリ、検索条件、トレンド分析の３セクション構成
5. 各セクションは折り畳み可能で、セクション名と文字数を表示
6. ユーザ登録機能を実装すること
7. ユーザ登録しないで利用する匿名モードを用意

### アセット

- React × Typescript
- データベースはFirestore
- デプロイはGitHub×Vercelで行う
